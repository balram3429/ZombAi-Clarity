LLMs hallucinate when they don't know. RAG gives them a memory — real documents, retrieved in real time.

🔍 Why RAG Exists

A standard LLM has frozen knowledge — it only knows what it was trained on, up to a cutoff date. Ask it about your company's Q4 report, your codebase, or yesterday's news — it either hallucinates or admits ignorance.

RAG solves this by splitting the problem: use a fast retrieval system to find relevant documents, then hand them to the LLM as context. The LLM doesn't need to memorize everything — it just needs to read and reason.

RAG vs Standard LLM

⚙ The RAG Pipeline — Step by Step

RAG has two phases: an offline indexing phase (done once) and an online retrieval+generation phase (done per query).

Full RAG Pipeline
📄

1. Document Loading

Ingest PDFs, web pages, databases, code files — any text source into the pipeline.

✂️

2. Chunking

Split documents into overlapping chunks (e.g. 512 tokens with 50-token overlap). Chunk size is a critical hyperparameter.

🔢

3. Embedding

Each chunk is converted into a dense vector (e.g. 1536 dimensions) using an embedding model like text-embedding-ada-002.

🗄️

4. Vector Store

Vectors are indexed in a vector database (Pinecone, Weaviate, Chroma, FAISS) for fast similarity search.

❓

5. Query Embedding

User's question is embedded with the same model — queries and documents live in the same vector space.

🔎

6. Similarity Search

Find top-k most similar chunks via cosine similarity or ANN search. Typically retrieve 3-10 chunks.

🤖

7. Augmented Generation

Retrieved chunks + original query are stuffed into the LLM's context window. The LLM generates a grounded answer.

🔢 Embeddings — Text as Coordinates

An embedding model maps text to a point in high-dimensional space. Similar texts are close together. This is what makes semantic search possible — not keyword matching, but meaning matching.

2D Embedding Space (simplified)

📐 Similarity Search

Given a query vector, we want the k nearest chunks. The most common metrics: cosine similarity (angle between vectors) and dot product. Vector databases like Pinecone use ANN algorithms to make this blazing fast even with millions of documents.

Live Similarity Ranking
cosine_similarity(A, B) = (A · B) / (|A| × |B|)
Range: -1 (opposite) to +1 (identical). RAG uses top-k by this score.

🚀 Advanced RAG Techniques

🔄

HyDE

Generate a hypothetical answer, embed it, then retrieve. Improves recall for abstract queries.

🔀

Re-ranking

Use a cross-encoder to re-score retrieved chunks for precision after initial retrieval.

🌳

Hierarchical Chunking

Store summaries at multiple granularity levels — paragraph, section, document.

🕸

GraphRAG

Build a knowledge graph over documents for multi-hop reasoning and relationship queries.
