Agentic AI

AI that plans, reasons, uses tools, and takes actions in the world. The frontier where models become actors.

🤖 What Makes an AI "Agentic"?

A standard LLM responds to a prompt. An agent operates in a loop: it receives a task, reasons about what to do, takes an action (calling a tool, running code, searching the web), observes the result, and iterates until the task is complete. The LLM is the brain; tools are its hands.

Agentic Loop Animation

🧠 ReAct — Reason + Act

ReAct (Reasoning + Acting) is the foundational pattern for agents. The model interleaves Thought → Action → Observation in a loop. This lets it break complex tasks into steps, correct errors, and use tools strategically.

🔧 Tools — The Agent's Capabilities

Tools are functions the agent can call. Each tool has a name, description, and input schema. The LLM decides which tools to call based on its reasoning. Tools can be anything from web search to code execution to database queries.

🕸 Multi-Agent Systems

Complex tasks are too big for one agent. Multi-agent systems use an orchestrator that delegates to specialist sub-agents. Each sub-agent has focused tools and expertise. The orchestrator synthesizes results into a coherent output.

🏷 Types of Agentic Systems

Tool-Use Agent
Function Calling
LLM that can call predefined tools. Examples: OpenAI function calling, Claude tool use, LangChain agents.
Autonomous Agent
AutoGPT / BabyAGI
Self-directed agent that breaks goals into tasks, executes them, and creates new tasks based on results.
Multi-Agent
Orchestrator + Specialists
LangGraph, CrewAI, AutoGen. One coordinator agent routes tasks to specialist agents with specific tools.
Computer Use
GUI Agent
Agent that sees a screen and controls mouse/keyboard. Claude Computer Use, Operator. Takes actions in real software.

📋 Planning in Agents

Advanced agents don't just react — they plan ahead. Planning techniques include Chain-of-Thought (step-by-step reasoning), Tree-of-Thought (exploring multiple reasoning branches), and task decomposition (breaking complex goals into subtasks).
